# 코드 리뷰

다음 항목들을 중점적으로 검토해주세요:

## 1. 보안
- XSS, SQL Injection 등 취약점
- 입력 검증 및 sanitization
- 인증/인가 로직 확인

## 2. 성능
- 비효율적인 루프나 쿼리
- 불필요한 DOM 재렌더링
- 메모리 누수 가능성

## 3. 코딩 표준
- Faculty-status 프로젝트 코딩 컨벤션 준수
- 변수명/함수명 명확성
- 주석 적절성 (특히 한글 데이터 처리 부분)

## 4. 에러 핸들링
- try-catch 적절한 사용
- 사용자 친화적인 에러 메시지
- 에러 로깅

## 5. 한글 데이터 처리
- UTF-8 인코딩 확인
- 한글 문자열 처리 안정성
- Excel 파싱 시 인코딩 문제

구체적인 개선 제안과 함께 코드 스니펫을 제공해주세요.
