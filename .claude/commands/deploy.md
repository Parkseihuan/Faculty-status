# 배포 체크리스트

배포 전 다음 항목들을 확인해주세요:

## 1. 코드 품질
- [ ] 모든 테스트 통과
- [ ] 코드 리뷰 완료
- [ ] 불필요한 console.log 제거
- [ ] TODO/FIXME 주석 확인

## 2. 데이터 처리
- [ ] 실제 교원 데이터로 파싱 테스트
- [ ] 한글 인코딩 문제 없음
- [ ] 날짜 형식 일관성 확인

## 3. 프론트엔드
- [ ] 모든 브라우저에서 테스트 (Chrome, Firefox, Safari)
- [ ] 반응형 디자인 확인
- [ ] 로딩 상태 UI 확인
- [ ] 에러 메시지 사용자 친화적

## 4. 백엔드
- [ ] API 엔드포인트 테스트
- [ ] 에러 핸들링 적절
- [ ] 데이터베이스 연결 안정성
- [ ] 인증/인가 동작 확인

## 5. 보안
- [ ] 환경 변수 (.env) 설정 확인
- [ ] 민감한 정보 코드에 없음
- [ ] CORS 설정 확인
- [ ] 관리자 페이지 인증 동작

## 6. 성능
- [ ] 큰 Excel 파일 파싱 성능 테스트
- [ ] 페이지 로딩 속도 확인
- [ ] 메모리 누수 없음

## 배포 절차

### 1. Git 커밋 및 푸시
```bash
git add .
git commit -m "설명적인 커밋 메시지"
git push origin main
```

### 2. Render.com 확인
- 자동 배포 시작 확인
- 빌드 로그 모니터링
- 배포 완료까지 대기 (2-3분)

### 3. 배포 후 검증
- 배포된 사이트 접속
- 주요 기능 동작 확인
- 에러 로그 확인

### 4. 롤백 준비
문제 발생 시:
```bash
# 이전 커밋으로 되돌리기
git revert HEAD
git push origin main
```

## 주의사항

- **브라우저 캐시**: 사용자들에게 Ctrl+Shift+R로 강력한 새로고침 안내
- **데이터베이스 마이그레이션**: 스키마 변경 시 기존 데이터 마이그레이션 필요
- **환경 변수**: Render.com 대시보드에서 환경 변수 확인

모든 체크리스트 항목이 완료되었는지 확인하고, 배포를 진행하세요.
